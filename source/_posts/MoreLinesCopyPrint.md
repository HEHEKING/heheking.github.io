呜呜呜，刷洛谷第一题，咱的想法和许多老哥是一样的：直接打印。但这种解法无疑是费时的，而且这算哪门子做题。

多行字符串的输入输出，个人认为难点在于换行符与空格，这玩意是剪切板没法标识的。于是想到替换，那么 base64 是最佳的选择，将多行字符串转化为 base64 编码，输出，完美解决。

但最近恰巧在学PHP，忽然有一个大胆的想法：PHP 的解释器在对待 `<? php ?>` 以外的内容时，会当作字符串直接输出。<!-- more -->利用这一特性，可以直接复制黏贴！

哈哈哈，这种通过应该算是漏洞，而且果不其然，题解中已经有人提到这种解法。

是时候直面 cp 问题了！

## Java

使用 Java 的过程中，确实没怎么接触到字符串输出。。。

但是辣鸡 Java 应该没办法定义一个多行文本，就算使用：

```java
public class Test {
	public static void main(String args[]) {
		String str = "                ********\n"
				+ "               ************\n"
				+ "               ####....#.\n"
				+ "             #..###.....##....\n"
				+ "             ###.......######\n"
				+ "                ...........";
		System.out.println(str);
	}
}
```

也是非常的木大，这和直接输出没有区别。况且如果需要操作，则需要识别出文本中的换行符，这显然是没有意义的。

上网查询了 Java 的多行字符串，得到的结果是 只有 Java 13 以上的版本有 多行字符串/文本块（Text Blocks）的 **预览特性**

```java 13
public class TestMultiLineStr {
    public static void main(String[] args) {
        String data = """
                      你好
                      世界
                      """;
        System.out.printf(data);
    }
}
```

资料来源：https://www.letianbiji.com/java/java13-text-block.html

## C++

看题解的时候翻到了这个，用的是 C++ 11 引入的新特性：原始字符串 raw string literal

语法为：`R"(...)"`

示例如下：

```c++
#include<iostream>
int main()
{
    std::cout<<R"(                ********
               ************
               ####....#.
             #..###.....##....
             ###.......######              ###            ###
                ...........               #...#          #...#
               ##*#######                 #.#.#          #.#.#
            ####*******######             #.#.#          #.#.#
           ...#***.****.*###....          #...#          #...#
           ....**********##.....           ###            ###
           ....****    *****....
             ####        ####
           ######        ######
##############################################################
#...#......#.##...#......#.##...#......#.##------------------#
###########################################------------------#
#..#....#....##..#....#....##..#....#....#####################
##########################################    #----------#
#.....#......##.....#......##.....#......#    #----------#
##########################################    #----------#
#.#..#....#..##.#..#....#..##.#..#....#..#    #----------#
##########################################    ############ )";
}
```
